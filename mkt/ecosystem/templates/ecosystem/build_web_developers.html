{{ _('For Current Web Developers') }}

{{ _('Minimum requirements') }}

{%- trans %} If you are a Web developer and you have a website or Web application that you would like to make into an installable Open Web app, there is technically very little that you need to do. The minimum requirements are few: {% endtrans -%}

  1. {%- trans url='//developer.mozilla.org/docs/Apps/Manifest' %} Create an app manifest. {% endtrans -%}
  2. {# L10n: Do not localize text in #} {%- trans url='//developer.mozilla.org/docs/Apps/Apps_JavaScript_API' %} Serve the app manifest in a file with a file extension of .webapp. Set the Content-Type header to application/x-web-app-manifest+json. {% endtrans -%}
  3. {%- trans url='//developer.mozilla.org/docs/Apps/Apps_JavaScript_API' %} Publish the app, either on your own site or in an app store (or both). Publishing it yourself requires adding some code to your site to manage installing and updating the app in users' browsers. {% endtrans -%}

{{ _('Optional features') }}

{%- trans url='//developer.mozilla.org/docs/Apps/Apps_architecture#Web_runtime' %} Philosophically, the idea of an installable Open Web app is much more than simply adding a manifest to your site. Web standards technologies can be viewed as a full-blown application platform that happens to use a browser engine for rendering user interfaces and interpreting code, and happens to use Web protocols for communicating with a server. Mozilla offers "Web runtime" executables for various platforms so that apps can run in their own window, without a browser window frame. {% endtrans -%}

{%- trans %} To "appify" a website, there are many application-specific questions to consider: {% endtrans -%}

  • {{ _('Should my app work when not connected to the Web?') }}
  • {%- trans %} How does my app use data, and how does it need to be stored? {% endtrans -%}
  • {%- trans url_workers='//developer.mozilla.org/docs/Using_web_workers', url_sockets='//developer.mozilla.org/docs/WebSockets' %} Can my app's performance benefit from advanced platform features like Web Workers or WebSockets? {% endtrans -%}
  • {{ _('And many more') }}

{%- trans %} If you want to take full advantage of the capabilities of installable apps, there is plenty that you can do. For example: {% endtrans -%}

  • {%- trans url='//developer.mozilla.org/docs/Web_Development/Responsive_Web_design' %} Use responsive Web design to look good and work well on all devices. {% endtrans -%}
  • {{ _('Charge money for apps.') }}
  • {%- trans url='//developer.mozilla.org/docs/Apps/Identity_integration' %} Provide a way to identify users. {% endtrans -%}
  • {%- trans url='//developer.mozilla.org/docs/Apps/Using_apps_offline' %} Enable offline caching so the app can be used when the device is not on the Internet. {% endtrans -%}
  • {%- trans url_idx='//developer.mozilla.org/docs/IndexedDB', url_storage='//developer.mozilla.org/docs/DOM/Storage' %} Store data locally using either IndexedDB or localStorage. {% endtrans -%}
  • {%- trans url='//developer.mozilla.org/docs/Apps/Platform-specific_details' %} Launch the app natively (with an icon on the desktop or the home screen). {% endtrans -%}
  • {%- trans url_geo='//developer.mozilla.org/docs/Using_geolocation', url_motion='//developer.mozilla.org/docs/DOM/Orientation_and_motion_data_explained' %} Use device APIs to interact with hardware, such as geolocation and orientation. {% endtrans -%}
  • {%- trans %} Give users a way to give you feedback. Mozilla's user research shows that users want to give feedback to app developers, and want to know that there is a human receiving it. They want to make suggestions and get help with problems. They may stop using an app if they have a problem and there is no way to get help with it. {% endtrans -%}
