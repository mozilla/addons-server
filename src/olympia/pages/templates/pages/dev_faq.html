{{ _('Developing an Add-on') }}

{% trans addons_link_open=''|safe, link_close=''|safe %}

How do I build an Add-on?

Mozilla provides documentation on how to build an add-on via the {{ addons_link_open }}Mozilla Developer Network{{ link_close }}.

{% endtrans %}

{{ _('What is a ".xpi" file?') }}

{% trans %} Extensions are packaged and distributed in ZIP files or Bundles, with the XPI (pronounced "zippy") file extension. {% endtrans %}

{{ _('What is XUL?') }}

{% trans %} XUL (XML User Interface Language) is Mozilla's XML-based language that lets you build feature-rich cross platform applications. It provides user interface widgets like buttons, menus, toolbars, trees, etc that can be used to enhance add-ons by modifying parts of the browser UI. {% endtrans %}

{% trans webext_link_open=''|safe, portlegacy_link_open=''|safe, link_close=''|safe %} Note: XUL is considered a legacy technology in Firefox, and add-ons using XUL will no longer load starting from Firefox 57. Please use {{ webext_link_open}}WebExtensions{{ link_close }} APIs to develop new add-ons. If you have a legacy add-on, consider migrating it to WebExtensions. For information on migrating a legacy add-on, please visit {{ portlegacy_link_open }}Porting a legacy Firefox add-on{{ link_close }}. {% endtrans %}

{{ _('How do I test for compatibility with the latest version of Mozilla software?') }}

{% trans %} To ensure compatibility with the latest Mozilla software, it's important to download updates as they become available and test your add-on to ensure that it is still functioning as expected. Since these releases at times introduce architectural changes that may impact the functionality of your add-on, it's important to be actively involved in the beta process to ensure that your add-on users are not negatively impacted upon final release of Mozilla software. {% endtrans %}

{{ _('Contributing your Add-on') }}

{{ _('Can I host my own add-on?') }}

{% trans amo_url='https://addons.mozilla.org' %} Yes. Many developers choose to host their own add-ons. Choosing to host your add-on on Mozilla's add-on site, though, allows for much greater exposure to your add-on due to the large volume of visitors to the site. {% endtrans %}

{{ _('Can Mozilla host my add-on?') }}

{% trans url='https://addons.mozilla.org' %} Yes. You can host your add-on on Mozilla's add-on website. {% endtrans %}

{{ _('What is AMO?') }}

{% trans %} Mozilla's AMO (https://addons.mozilla.org) is the incubator that helps developers build, distribute, and support fantastic consumer products powered by Mozilla. It provides you the tools and infrastructure necessary to manage, host and expose your add-on to a massive base of Mozilla users. {% endtrans %}

{{ _('Does Mozilla keep my account information private?') }}

{% trans url='https://www.mozilla.org/en-US/privacy/websites/' %} Yes. Our Privacy Policy describes how your information is managed by Mozilla. {% endtrans %}

{{ _('What are the "developer tools" listed on AMO?') }}

{% trans %} The "Developer Tools" dashboard is the area that provides you the tools to successfully manage your add-ons. It provides the functionality necessary to submit your add-ons to AMO, manage add-on information, and review statistics. {% endtrans %}

{{ _('Does Mozilla have a policy in place as to what is an acceptable submission?') }}

{% trans policy_link_open=''|safe, guide_link_open=''|safe, link_close=''|safe %} Yes. Mozilla's {{ policy_link_open }}Add-on Review Policies{{ link_close }} describe what is an acceptable submission. This policy is subject to change without notice. In addition, the Add-on Reviewer Team uses the {{ guide_link_open }}Add-on Review Guide{{ link_close}} to ensure that your add-on meets specific guidelines for functionality and security. {% endtrans %}

{{ _('How do I submit my add-on?') }}

{% trans review_policy_link_open=''|safe, link_close=''|safe %} The Developer Tools dashboard will allow you to upload and submit add-ons to AMO. You must be a registered AMO user before you can submit an add-on. Before submitting your add-on be sure to you have read the {{ review_policy_link_open }}Add-on Review Policies{{ link_close }} to ensure that your add-on has met our guidelines. {% endtrans %}

{{ _('What operating system do I choose for my add-on?') }}

{% trans %} You must choose the operating systems on which your add-on will successfully function. {% endtrans %}

{{ _('What category do I choose for my add-on?') }}

{% trans %} The choice of category is dependent on what type of audience you are targeting and the functionality of your add-on. If you're unsure of which category your add-on falls into, please choose "Other". The AMO team may re-categorize your add-on if it's determined that it's better suited in a different category. {% endtrans %}

{{ _('Can I specify a license agreement for using my add-on?') }}

{% trans %} Yes. You can specify a license agreement when submitting your add-on. You can also add or update a license agreement via the Developer Tools dashboard after your add-on has been submitted. {% endtrans %}

{{ _('Can I include a privacy policy for my add-on?') }}

{% trans %} Yes. You can specify a privacy policy when submitting your add-on. You can also add or update a privacy policy via the Developer Tools dashboard after your add-on has been submitted. {% endtrans %}

{{ _('Add-on Review Process') }}

{{ _('Must my add-on be reviewed?') }}

{% trans url='https://wiki.mozilla.org/Add-ons/Reviewers/Guide' %} New and updated add-ons may be suject to reviewed to ensure that Mozilla users have a stable and safe experience. The guidelines outlined in the Add-on Review Guide are used to review add-on submissions. {% endtrans %}

{{ _('Who reviews my add-on?') }}

{% trans url='https://wiki.mozilla.org/Add-ons/Reviewers/Guide' %} Add-ons are reviewed by the Add-on Reviewers, a group of talented developers that volunteer to help the Mozilla project by reviewing add-ons to ensure a stable and safe experience for Mozilla users. When communicating with a reviewer, please be courteous, patient and respectful as they are working hard to ensure that your add-on is set up correctly and follows the guidelines outlined in the Add-on Review Guide. {% endtrans %}

{{ _('What are the guidelines used to review my add-on?') }}

{% trans url='https://wiki.mozilla.org/Add-ons/Reviewers/Guide' %} The Add-on Reviewer Team follows the Add-on Review Guide when testing an add-on. It is important that add-on developers review this guide to ensure that common problem areas are addressed prior to submitting their add-on. This will greatly assist in avoiding review issues. {% endtrans %}
